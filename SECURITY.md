# 安全指南

## 项目定位

OpenMicroChat 是一个**学术研究与技术探索**项目，用于研究即时通讯系统的交互设计与前端架构。本项目**不适合直接用于生产环境或大规模用户服务**。

## 安全声明

- 本项目**未经过专业安全审计**
- 通信内容**未实现端到端加密**
- 认证系统为基础 JWT 实现，不包含多因素认证、速率限制等生产级安全措施
- 数据存储使用单文件 JSON 数据库（lowdb），不适合高并发或大数据量场景

## 私有化部署建议

如果你希望使用本项目进行通讯，**强烈建议私有化部署**：

1. **不要使用公共服务器**存储敏感聊天内容
2. 部署时**务必修改** `.env` 中的 `JWT_SECRET` 为随机强密码
3. 使用 **HTTPS**（TLS）保护传输层安全
4. 限制服务器的网络访问范围（防火墙规则）
5. 定期备份数据文件（`db.json`）
6. 不要在公开网络暴露管理接口

## 已知安全限制

| 领域 | 现状 | 生产环境需要 |
|------|------|-------------|
| 传输加密 | 依赖部署者配置 HTTPS | TLS 1.3 强制 |
| 消息加密 | 明文存储 | 端到端加密 (E2EE) |
| 认证 | 基础 JWT | MFA + 速率限制 + 会话管理 |
| 数据库 | lowdb (JSON 文件) | PostgreSQL/MySQL + 加密存储 |
| 内容审核 | 无 | 敏感词过滤 + 人工审核 |
| 日志审计 | 基础 console.log | 结构化日志 + 审计追踪 |
| 密码策略 | 无复杂度要求 | 最小长度 + 复杂度 + 泄露检测 |

## 漏洞报告

如果你发现安全漏洞，请通过 GitHub Issues 提交（公开项目，无需私密通道）。由于本项目为学术研究性质，修复时间不做 SLA 承诺。

## 数据处理

- 本项目不主动收集任何用户数据
- 所有数据存储在用户自行部署的服务器上
- 客户端数据存储在浏览器 localStorage 中
- 用户可随时清除本地数据（浏览器设置 → 清除站点数据）
- 服务端数据可通过删除 `db.json` 文件完全清除
