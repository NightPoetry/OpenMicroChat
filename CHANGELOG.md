# Changelog

本文件记录 OpenMicroChat 的开发历程与版本变更。

## v1.0.0 (2026-05-01)

首个公开发布版本。

### 项目初始化

- 搭建基于 Express + lowdb 的后端服务，实现 JWT 认证、消息收发、联系人管理、群组管理等 REST API
- 搭建基于原生 ES Modules 的前端架构，采用发布-订阅模式的状态管理和事件总线实现模块间通信
- 实现 Socket.io 实时消息通信
- 实现体验模式，提供预设数据供用户免注册体验

### 前端模块开发

- 认证模块：登录、注册、Token 管理、用户菜单
- 联系人模块：联系人列表渲染、搜索过滤、添加好友
- 聊天模块：消息发送与接收、多媒体支持（图片/语音/文件）、动态聊天头部
- 设置模块：上下文感知的设置页面（根据联系人类型动态渲染用户/好友/群聊设置）
- 标签模块：标签胶囊栏、标签选择与创建、韦恩图交集动画
- 韦恩图可视化：SVG 渲染、弹性缩放动画序列、区域点击交互筛选

### 交互设计

- 长按联系人触发标签管理面板，支持就地新建标签
- 选中一个标签展示单圆动画，选中两个标签展示韦恩图交集动画
- 韦恩图三区域（仅A、交集、仅B）可点击进行子筛选
- 联系人名称旁显示彩色标签圆点
- 移动端侧边栏滑动切换，返回按钮自动显隐

### 样式与布局

- 采用 CSS 自定义属性实现主题系统，支持浅色/深色模式切换
- 应用防御性布局原则：flex 容器 `min-height: 0` 防溢出、文本截断保护
- 响应式设计：桌面双栏、平板收窄、手机单栏三档适配
- 添加 safe-area 适配，支持刘海屏和底部横条设备
- 无障碍支持：`focus-visible` 焦点样式、`prefers-reduced-motion` 动画降级

### 设计文档

- 编写设计哲学文档，记录触控优先、标签即视图、操作语义统一等核心原则
- 编写设计演进文档，记录从原型到当前形态的五个阶段决策过程
- 编写技术架构文档，记录分层结构、模块通信事件表、API 路由表

### 多平台支持

- 集成 Tauri 2，支持打包为 macOS (.app/.dmg)、Windows (NSIS)、Linux (AppImage/deb) 桌面应用
- 配置 GitHub Actions CI/CD，推送版本标签自动触发三平台构建与 Release 发布
- 配置 Android APK 构建流程
- 生成多尺寸应用图标（PNG/ICNS/ICO）

### 服务器配置

- 实现可配置的服务器地址功能，统一 API 调用层
- Tauri 环境自动显示服务器地址输入框，Web 环境默认使用同源地址
- 服务器地址持久化到 localStorage，用户菜单提供服务器设置入口
